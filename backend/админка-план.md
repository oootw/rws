# Админ-панель — план для следующей сессии

Документ-handoff. Цель: следующая сессия (в т. ч. с холодным контекстом)
должна продолжить с того места, где закончили, без перечитывания всего чата.

Связанные документы (читать сначала):
- `backend/саммари.md` — общая шпаргалка по backend и DDD-структуре проекта.
- `backend/docs/architecture.md` — правила слоёв, naming, тестирование.
- `bootstrap/providers.php` — порядок сервис-провайдеров.

## 1. Архитектурный контракт админки (фиксировано)

- **Стек:** Filament **v5.6** (`composer require filament/filament:^5.6`).
- **URL:** `/admin` на основном домене (см. `config/guardreviews.php` → `admin.path`).
- **Auth:** один супер-админ из `.env` (`ADMIN_EMAIL` + `ADMIN_PASSWORD_HASH`), отдельный гард `admin`, кастомный `UserProvider` (без таблицы `admin_users`).
- **DDD-граница:** Filament Resource = Interface-слой, такой же тонкий как HTTP-контроллеры.
  - **Чтение** (table/infolist) — напрямую через Eloquent (read-model паттерн).
  - **Запись** (create/edit/delete/custom actions) — **только через Application use cases**. Eloquent::save() в обход домена запрещён.
  - **Исключение:** конфигурационные сущности без сложных инвариантов (Tariff) — CRUD напрямую через Eloquent (`CreateRecord`/`EditRecord` без override); use case заводим **только** под инварианты (например, `SetDefaultTariff` гарантирует «ровно один is_default»).
- **Имперсонация:** в Фазе 1 = выпуск Sanctum PAT с ability `impersonated` и TTL 15 мин. Plain-text токен показывается в Filament Notification один раз. Будущий Owner PWA подцепится через endpoint `POST /api/auth/exchange-impersonation-token` (он ещё не реализован — это задача Owner PWA).

## 3. Подводные камни (узнаны на горьком опыте — не наступать заново)

### 3.1. Окружение разработки

- **Локальный PHP 8.2 на хосте**, проект требует **8.4**. Все команды (`composer`, `artisan`, тесты) запускать в Docker:
  ```bash
  docker run --rm -v $(pwd)/backend:/app -w /app php:8.4-cli-alpine sh -c \
    "apk add --no-cache icu-libs icu-dev libzip-dev zip > /dev/null 2>&1 && \
     docker-php-ext-install intl > /dev/null 2>&1 && \
     php artisan test 2>&1"
  ```
- Для `composer require` можно с хоста: `composer require <pkg> --ignore-platform-req=php --ignore-platform-req=ext-intl` (composer обновит lock, реальный install через `composer install` пойдёт в контейнере при сборке Docker-образа).
- **Warnings `file_get_contents(/app/.env): Failed to open stream`** — общесистемный шум при одноразовых `docker run` (нет `.env` в контейнере). Тесты от этого не падают; assertions работают.

### 3.2. Filament v5 — особенности

- **Filament v3 НЕ работает с Laravel 13** (`illuminate/auth ^10|^11|^12`). Используем v5.x (`illuminate/contracts: ^11.28|^12.0|^13.0`).
- **Авто-discovery resources/pages/widgets** — в `AdminPanelProvider->panel()` указаны пути; вложенные папки сканируются рекурсивно.
- **`$model = User|Place|Review|Tariff|PaymentTransaction::class`** в Resource. Домен (например, `App\Domain\Iam\Owner`) для Filament-читателя не виден — он работает с persistence-моделью.
- **Generator** `php artisan make:filament-resource` создаёт глубокую структуру: `XxxResource.php` + `Tables/`, `Schemas/`, `Pages/`. Мы перезаписываем сгенерённые файлы по нашему шаблону (см. OwnerResource как эталон).
- **`canCreate()=false`** на ресурсе скрывает `/create`-маршрут и `CreateAction` (см. `PaymentTransactionResource`). Достаточно вернуть `false`; `getPages()` без `'create' =>` страницу не выставит.
- **`Action::modalContent(fn ($record) => view(...))`** — модалка с произвольным blade (используем для preview_qr).
- **`Heroicon` enum** покрывает все стандартные outlined-иконки: `Heroicon::OutlinedUsers/MapPin/ChatBubbleBottomCenterText/CurrencyDollar/Banknotes` и т. п.

### 3.3. Кастомный гард `admin` — заглушки в `AdminUser`

`AdminUser extends GenericUser` нужен набор «Eloquent-подобных» методов, которых GenericUser не имеет — но Filament/Livewire зовут их в blade-view:
- `getKey(): string` — иначе `notifications.blade.php` падает.
- `getAttributeValue(string $key): mixed` — иначе `avatar/user.blade.php` падает.
- `getFilamentName()` + `getFilamentAvatarUrl()` через `HasName`/`HasAvatar` интерфейсы.
- `getRememberToken(): ?string` + `setRememberToken($v): void` — overriden чтобы не получать `Undefined array key "remember_token"` при logout.

### 3.4. Тесты Livewire-компонентов Filament

Чтобы тестировать таблицы/фильтры через `Livewire::test(ListXxx::class)`, нужно:
```php
use Filament\Facades\Filament;

beforeEach(function (): void {
    // ... actingAs admin ...
    Filament::setCurrentPanel('admin');
    Filament::setServingStatus(true);
});
```
Иначе `NoDefaultPanelSetException`.

### 3.5. Что НЕ работает / на что наступали в Filament v5

- **`Action::make('reset_filters')->visible(false)` как placeholder в `toolbarActions`** — Filament всё равно пытается зарезолвить и падает. Если actions нет — передавай пустой массив `->toolbarActions([])`.
- **TernaryFilter с вложенным `where(closure)`** — падает `Call to a member function newQueryWithoutRelationships() on null`. Использовать плоскую форму `->queries(true: fn $q, false: fn $q)`. См. `OwnersTable::subscription_status` / `PlacesTable::is_active`.
- **`->modifyQueryUsing(fn (Builder $q) => $q->with('tariff'))`** — лишний. Filament сам определяет нужный `with` из колонки `tariff.title`.
- **Repeater хранит ключи как UUID** (`array<string, array{...}>`) — use case ожидает `list`. Нормализуем через `PlaceForm::normalizeRepeaterPlatforms($data['platforms'])` в обоих Page-классах.
- **`disabledOn('edit')`** в форме делает поле видимым, но не дегидратирует — нормально для «зафиксированных» полей (Place.user_id).

### 3.6. PaymentTransaction migration / FK

Таблица `payment_transactions`:
- `user_id` → `users.id` (CASCADE on delete) — каскад работает.
- `tariff_id` → `tariffs.id` (**restrict on delete**) — обязательно нужен при создании; нельзя удалить тариф, если на него есть транзакции.

В тестах: при создании транзакции для проверки каскада удаления Owner — обязательно передавать `tariff_id`.

### 3.7. `phpunit.xml` APP_KEY

Без `APP_KEY` Filament падает в `EncryptCookies`. Зафиксировано тестовое значение:
```xml
<env name="APP_KEY" value="base64:AckfSECXIvnK5r28GVIWUAxmbBSjTsmF11WBzy3uoaA="/>
```

### 3.8. Default-тариф и обратная совместимость

- До Фазы 4 «default» определялся по `title='MVP'` в `EloquentTariffRepository::DEFAULT_TITLE`. После Фазы 4 — по boolean колонке `is_default`. **Миграция backfill'ит MVP=true**, `TariffSeeder` тоже выставляет `is_default => true`. Тесты, ранее опиравшиеся на seeded MVP, не сломались.
- `TariffFactory` намеренно **не** ставит `is_default=true` — иначе любая фабрика создавала бы «второй default». В тестах, где нужен default, либо `$this->seed(TariffSeeder::class)`, либо явно `is_default => true` в `Tariff::factory()->create([...])`.

### 3.9. Refire webhook для платежа

`PaymentTransactionActionFactory::refireWebhook()` строит payload руками:
```php
['TerminalKey', 'OrderId' => $record->id, 'Status' => 'CONFIRMED'|'REJECTED',
 'Success' => bool, 'Amount' => int, 'PaymentId' => $external_id ?? "admin-refire-{$id}"]
```
Подписывает через `TinkoffTokenSigner::sign($payload, $config->secretKey())` и прогоняет
через `HandlePaymentNotificationHandler` — путь идентичен реальному webhook'у,
включая идемпотентность и продление подписки. Action скрыт на production
(`->visible(fn () => ! app()->isProduction())`).

## 7. Чек-лист «добавить новую фичу в админке»

Всякий раз, когда добавляешь новый Resource или action:

1. **Domain.** Если новые мутации — добавить методы в агрегат + расширить Repository (`delete`/`findAll`/…).
2. **Application.** Создать папку `<UseCase>/` с `Command` + `Handler` (+ `Result`/`View` при необходимости). Если есть общий хелпер ввода (как `PlatformsBuilder`) — положить в `Application/<Context>/Support/`.
3. **Infrastructure.** Если новый порт — реализовать адаптер + bind в `<Context>ServiceProvider`.
4. **Filament Resource.**
   - `$model = <EloquentModel>::class`.
   - `table()/form()/infolist()` — конфиг.
   - `Actions/<Context>ActionFactory.php` — custom actions.
   - `Pages/Create*.php` + `Edit*.php` — переопределить `handleRecordCreation` / `handleRecordUpdate` → use case. **Исключение**: конфигурационные сущности без сложных инвариантов (Tariff) — оставить дефолтные `CreateRecord`/`EditRecord`.
   - `DeleteAction::make()->action(fn ($record) => app(DeleteHandler::class)->handle(...))`.
   - Production-safety: actions с побочными эффектами наружу (webhook, тестовая отправка) — `->visible(fn () => !app()->isProduction())`.
5. **Тесты.**
   - Unit на handlers (+ Domain, если методы добавлены).
   - Feature на Resource:
     - HTTP smoke (list/create/view/edit/canCreate если read-only).
     - Livewire-фильтры (через `Livewire::test(ListPage::class)`).
     - Интеграция handlers поверх Eloquent (`RefreshDatabase`).
     - Edge cases: cascade-delete, idempotency, validation errors.
6. **Pint.** `./vendor/bin/pint <new-files>`.
7. **Проверь** что все 4 слоя на месте: Domain нет `use Illuminate\…`, FQ-классы в Application импортированы, контроллеры/actions тонкие.
